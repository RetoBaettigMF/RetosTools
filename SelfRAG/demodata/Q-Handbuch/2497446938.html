Nummerierungssystem:

Für die Nummerierung von Projekten, Subprojekten, Offerten/Aufträgen, Rechnungen, Kreditoren und Debitoren existiert folgendes Schema:

#PPPP.SSS.AA.RRv

Wobei

  • PPPP = Projektnummer
  • SSS = Subprojektnummer (entspricht meistens einer Phase eines Projektes, z.B. Analysephase, Implementierungsphase etc.)
  • AA = Offertnummer = Auftragsnummer (nach Bestellung der Offerte)
  • RR = Rechnungsnummer
  • v = optionale Versionsnummer

Beispiele:

  • #8113.002.03.06
  • #0032.001.01.02b


Die Projektnummer wird in folgendem File reserviert: "G:\Geteilte Ablagen\MF_Projekte\Projektnummern.xlsx"

Optionale Versionsnummer

Da möglicherweise Angebote oder Rechnungen überarbeitet werden müssen, sollen die Angebote und Rechnungen mit einer Versionsnummer versehen werden. Dabei wird die Versionsnummer nur dann erhöht, wenn der Kunde eine neue Version erhält.
Bei der ersten Version darf das <a> auch weggelassen werden; d.h. wenn keine Versionsnummer enthalten ist, dann ist damit die erste Version gemeint.

Beispiele:

  • #8113.002.03a oder #8113.002.03 => 1. Angebot
  • #8113.002.03b => 2. Angebot
  • #8113.002.03c => 3. Angebot
  • #8113.002.03.05a oder #8113.002.03.05 => 1. Rechnung
  • #8113.002.03.05b => 2. Rechnung
    #8113.002.03.05c =>3. Rechnung

Dokumentenpräfix

Um Dokumente voneinander zu unterscheiden, wird zusätzlich ein Präfix verwendet:

Präfix#PPPP.SSS.AA.RRv

Beispiele:

  • O#8113.002.03 3. Angebot für Subprojekt 002 in Projekt 8113
  • R#8113.002.03.02 Dazugehörige zweite Rechnung
  • KR#8113.002.04.01 Erste Kreditoren-Rechnung von der vierten Bestellung von einem Sublieferanten in diesem Subprojekt


Folgende Präfixe werden verwendet

  • Debitoren-Bezogen bzw. im Kontakt mit Kunden von uns:
    • O: Offerte an Kunde
    • B: Bestellung vom Kunden
    • A: Auftragsbestätigung an Kunde
    • R: Rechnung an Kunde
  • Kreditoren-Bezogen bzw. im Kontakt mit Lieferanten und Sublieferanten von uns:
    • KO: Offerte von Lieferant (Kreditoren-Offerte)
    • KB: Bestellung von uns an Lieferanten (Kreditoren-Bestellung)
    • KA: Auftragsbestätigung vom Lieferanten (Kreditoren-Auftragsbestätigung)
    • KR: Rechnung vom Lieferanten (Kreditoren-Rechnung)

Projektnummer PPPP

Für die Projektnummern gibt es folgendes Vergabesystem.

  • "G:\Geteilte Ablagen\MF_Projekte\Projektnummern.xlsx" öffnen
  • Neue Projektnummer eintragen
  • Datei speichern und schliessen

Subprojektnummer SSS

Ein Projekt hat immer mindestens ein Subprojekt. So ist z.B. #1234.001 das erste Subprojekt des Projektes #1234

Subprojekte fassen einen logisch zusammenhängenden Teil eines Projektes zusammen. Bei Fixpreis-Projekten z.B. Konzeptphase oder Implementierung. Bodyleasing-Projekte, welche über mehrere Jahre dauern und z.T. mehrere Bestellungen vom Kunden haben, haben in der Regel pro Jahr ein eigenes Controlling bzw. ein eigenes Subprojekt.

Die monatlichen Controllings werden für jedes Subprojekt einzeln erstellt.

Für die Vergabe der Subprojektnummern ist der Projektleiter zuständig.

Beispiele:

  • Projekt #1234 – Geberit Prüfstand XY
    • Subprojekt #1234.001 – Konzeptphase
    • Subprojekt #1234.002 – Implementierung
    • Subprojekt #1234.003 – Erweiterung XYZ
    • Subprojekt #1234.004 – Sammler für Kleinaufträge 2020
  • Projekt #2345 – Schott Bodyleasing
    • Subprojekt #2345.001 – Bodyleasing 2020
    • Subprojekt #2345.002 – Bodyleasing 2021

Offert- bzw. Auftragsnummer AA

(Bemerkung: Bis mitte 2020 wurden Subprojekte Aufträge genannt. Eine zusätzliche Beleg-Nummer wurde für Angebot, Bestellung, Auftrags-Bestätigung, und Rechnung verwendet. Neu gibt es keine separate Belegnummer mehr)


Um Missverständnissen vorzubeugen, werden wir den Ausdruck ‘Subprojekte’ für die erste Unterteilung der Projekte verwenden.

Der Ausdruck Auftrag wird erst nach bestätigtem Angebot (=Bestellungseingang) verwendet. Die Auftragsnummer entspricht der entsprechenden Offertnummer

Es gillt immer: Auftragsnummer == Offertnummer 


Für die Vergabe der Auftragsnummern ist der Projektleiter zuständig.

Beispiele:

  • #8113.002.03 => 3. Auftrag bzw. 3. Offerte für Subprojekt #8113.002


Rechnungsnummer RR

Da ein Auftrag mehrere Rechnungen zur Folge haben kann, werden bei Rechnungen, Gutschriften und Lieferscheinen zwingend noch 2 zusätzliche Ziffern angehängt.

Beispiel:

  • #8113.002.03.06 => 6. Rechnung von Projekt 8813, Subprojekt 002, Auftrag 03
