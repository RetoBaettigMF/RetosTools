Link zum Tech Radar

Zweck

Mit dem Tech Radar werden folgende Ziele verfolgt:

Identifikation der Schlüsseltechnologien der Cudos

  • Welche Technologien, Tools etc. sollen verwendet werden?

  • Welche Technologien sollen nicht mehr verwendet werden?

  • Sofern nicht von extern vorgegeben

Ideenlieferant für die Weiterbildung der Mitarbeiter

  • Welche Technologien könnten der Cudos von nutzen sein?

Identifikation von Know-How Trägern

  • Wer kann mir zu eine Technologie Auskunft geben oder mich beraten?

Gültigkeit

  • Jeder Mitarbeiter kennt den Tech Radar

  • Der Tech Radar soll befolgt werden sofern

    • Vom Kunden/Projekt nicht etwas anderes vorgegeben wurde.

    • Es auch sonst keine zwingende Gründe gibt, davon abzuweichen.

Zuständigkeiten

Verantwortlich für den Tech Radar ist der CTO. (Siehe auch https://mfcloud.atlassian.net/wiki/spaces/QHAN/pages/8340701188/CTO#Tech-Radar)

Zur Pflege des Tech Radars wird vom CTO 1-2x jährlich das Tech Radar Meeting einberufen. Teilnehmer an diesem Meeting sind alle https://mfcloud.atlassian.net/wiki/spaces/QHAN/pages/8340701188/CTO#Tech-Champions sowie interessierte Engineers

Repository

Der Tech Radar basiert auf einem Azure DevOps Projekt. Änderungen/Ergänzungen erfolgen per Branch und Pull-Request.
