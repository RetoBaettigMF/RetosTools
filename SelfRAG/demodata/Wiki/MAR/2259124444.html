Marketing : Excel Import ins CRM (Massenbearbeitung)

Created by Rachel Blaser, last modified by Reto Bättig on Juni 14, 2024

Wenn beispielweise die “Unsubscribes” von Mailchimp-Kampagnen im CRM nachgeführt werden müssen, kann dies über ein Excel bzw. CSV Import gemacht werden.

Hans Zemp kann den Import durchführen oder unterstützen.

Folgende Pflichtfelder braucht es im Excel für den Import:

  • Vorname

  • Nachname

  • primäre E-Mail

  • Letter Salutation

  • Zu importierende Felder (hinzufügen oder überschreiben)

→ Excel als CSV speichern (per Default wird “;” als Trennzeichen verwendet)

ACHTUNG: Der Import läuft nicht zuverlässig:

  • Zuerst Backup erstellen

  • Erste Tests mit wenigen Daten versuchen

  • Wenn in den Feldern Kommas oder Strichpunkte vorkommen, funktioniert CSV nicht mehr richtig. In diesem Fall folgendermassen vorgehen:

    • In Excel eine zusätzliche Seite machen

    • Alle Felder verweisen auf originalfelder und mit ““““ vorne und hinten ergänzen:
      =""""&Worksheet!A1&""""

    • Somit erscheinen Anfürhungszeichen um die Zellen

    • Dann dieses Sheet als .CSV speichern

    • Im Editor sieht man dann Drei hochkommas (“““)

    • Im Texteditor suchen und ersetzen “““ => “

    • Dann importieren

Import ins CRM

  1. Kommt das Excel oder .csv von Mailchimp:

    1. sind die Umlaute falsch dargestellt. Sie müssen zuerst manuell berichtigt werden.

    2. ist die Titelzeile Englisch (“Last Name”) und muss auf Deutsch (“Nachname”) umgestellt werden.

  2. Login V-Tiger CRM (Credentials siehe IT Passwörter)

  3. Reiter “Personen” auswählen (Kopfzeile)

  4. “Aktionen”

  5. “Import”

  6. Datei auswählen, Trennzeichen Semikolon (Default), hat Überschriften, Zusammenführen auswählen, Schritt 3: Keine weiteren Felder als “Vorname”, “Nachname” wählen:

  7. Wenn Zuordnung nicht automatisch gemacht wird (z.B. Organisation => Ohne), Zuordnung manuell erstellen

  8. Die manuelle Zuordnung kann unter einem Namen abgelegt werden (“speichere Zuordnung”), “kompletten Import ablehnen, …” ist eine Sicherheitsoption, die zum Zug kommt, ob sie ausgewählt ist oder nicht…

  9. “Import”

  10. Nach dem Import sollte folgendes Menu erscheinen:

  11. “Anzahl neu erstellter Datensätze” muss “0” sein, “Anzahl überschriebener Datensätze” muss 0 sein, “Anzahl aller zusammengeführten Datensätze” muss der Anzahl Records im .CSV File entsprechen.

  12. Mit “Fertigstellen” wird der Import abgeschlossen.
