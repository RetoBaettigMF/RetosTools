Der “Backup für Server” ist für den Restore auf einem lokalen Server gedacht.

Der “Backup für Cloud” ist für einen Restore in der Cloud gedacht. Dabei ist zu bedenken, dass in diesem Fall beim Restore die Gruppen (Rollen, Berechtigungen) NICHT restored werden (Stand 22.2.22). Der Grund ist Folgender: Wenn der Restore auf in einer anderen Destination (URL) gemacht wird, wäre das eine “Migration”, bei der die Berechtigungen eventuell anders sein müssten (Atlassian). Laut Atlassian soll es in zukünftigen Releasen eine Option “preserve group membership” geben, um beim Restore wieder die gleichen Berechtigungen zu haben wie vorher.

Um zur Zeit nach dem Restore die gleichen Berechtigungen zu haben gibt es zwei Möglichkeiten:

  • Die Berechtigungen manuell wieder herstellen

  • Den Umweg über einen lokalen Server machen:

    • Restore auf einen lokalen Server

    • Migration in die Cloud

ACHTUNG:

  • Da unsere Daten > 200MB sind, muss vor dem Restore zuerst bei Atlassian eine spezielle Bewilligung eingeholt werden, dass der Restore möglich ist.

  • Mit einer free/trial ID kann zwar der Restore durchgeführt werden, die Daten (Bereiche / Spaces) werden aber nicht angezeigt.

    • Umgehung: Trial upgrade production (14 Tage gültig)
