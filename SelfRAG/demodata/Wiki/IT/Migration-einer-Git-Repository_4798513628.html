IT : Migration einer Git-Repository

Created by Ehemaliger Benutzer, last modified on Jan. 25, 2018

1 Allgemein

1.2 - Erstellen eines Klones für die Migration

Der Klon für die Migration darf keine Arbeitskopie sein, sondern muss auch diejenigen Informationen enthalten, welche normalerweise nicht in einer lokalen Kopie enthalten sind. Dies kann mit dem Befehl

Git clone – mirror

erreicht werden. Dabei wird die sogenannte „bare repository“ kopiert.


Beispiel GitHub:

$ git clone --mirror https://github.com/wannefab/Test.git
Cloning into bare repository 'Test.git'...
remote: Counting objects: 30, done.
remote: Compressing objects: 100% (16/16), done.
remote: Total 30 (delta 7), reused 30 (delta 7), pack-reused 0
Unpacking objects: 100% (30/30), done.

1.2 - Erstellen einer leeren Repository auf dem Zielserver

Nun muss auf dem Server eine neue Repository angelegt werden. Diese soll komplett leer sein und z.B. nicht mit einem README initialisiert werden.

1.3 - Klon auf den Zielserver laden

Für das hochladen des Klones auf die Zielplatform muss wider die „—mirror“ option verwendet werden:

git push --mirror

Beispiel mit GitHub:

$ git push --mirror https://github.com/wannefab/Test.git

Compressing objects: 100% (21/21), done.
