IT : Anleitung für inStep Migration

Created by Reto Bättig, last modified by nizami.sevindi on Aug. 26, 2024
  • Projekt im inStep öffnen
  • Links auf "Alle Dateien/Produkte" clicken
  • Rechts auf das oberste Verzeichnis klicken (mit einer blauen Kugel markiert: 
  • => letzte Version holen anclicken und Daten in ein lokales Verzeichnis (z.B. c:\inStep) kopieren
    • Sicherstellen, dass "rekursiv" angeklickt ist
  • Auf [OK] Clicken
  • Warten, bis alle Daten kopiert wurden
  • Wichtig: bei einem Fehler werden unten in inStep Fehlermeldungen angezeigt. Im Fehlerfall zu einem Projektleiter gehen, der unterstützt!
  • Wenn alles ok ist, sollte es ca. so aussehen unten im inStep:
  • Das lokale Verzeichnis, wo es ausgecheckt wurde bei Bedarf umbenennen gemäss folgendem Schema:
    • KUNDE - Projekt - Projektnummer
  • Das lokale Verzeichnis verschieben nach G:\Geteilte Ablagen\Projekte
  • Kurz einige Stichproben machen, ob alle Dateien nun auf G: vorhanden sind
  • in Tabelle eintragen, dass Migration erfolgreich gelungen ist: https://docs.google.com/spreadsheets/d/1EzoFPMJA8d367DVPmc5tlg5zNg1o2912hq6Dfp30Q1w/edit#gid=0

  • in InStep neues Verzeichnis erstellen mit Namen "Neu bitte auf GDrive arbeiten"

  • Unterverzeichnisse in dieses neue Verzeichnis verschieben in inStep
