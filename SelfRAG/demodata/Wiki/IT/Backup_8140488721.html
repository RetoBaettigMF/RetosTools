Konzept

Alle Daten des Unternehmens werden auf Google Drive gespeichert. Die Daten werden entweder von anderen Services gepusht oder der Backup Script Server lädt sie von den anderen Servern herunter. Das Kopieren kann auch manuell geschehen. Bei automatischen Lösungen muss ein Monitoring eingebaut werden (automatisch oder Manuell).

Der Backup PC erzeugt periodisch (alle 1h, Minute '50) Snapshots auf Filesystem Ebene (btrfs). Diese sind jeweils mit einer Retention-Period versehen.

Der Backup PC muss möglichst autonom sein:

  • Keine automatischen Updates

  • Möglichst kein Zugriff von aussen

  • Firewall restriktiv einstellen

Falls Remote Administration verwendet wird: (Z.Bsp. Backup Server in einem Rechenzentrum)

  • Zugriff nur von dedizierten Rechnern, welche sonst nicht verwendet werden. Z.Bsp: Altes Notebook.

  • SSH mit Private Key. Key nur auf dediziertem Rechner und offline Speichermedium (z.Bsp. USB Stick)

Dadurch soll verhindert werden, dass bei einer Kompromittierung der Haupt-IT die Backup-Server mit betroffen sind.

Das Backup wird mit Hilfe von Influx/Grafana überwacht. Dies läuft auf einem Hetzner Server. Alle Vorgänge/Scripts senden an InfluxDB wenn sie ausgeführt wurden, und Grafana Alerting-Rules überwachen, ob das auch stattgefunden hat.

Source Code, inkl Dokumentation zum Aufsetzen: https://dev.azure.com/MFEngineeringAG/_git/M_F Backup

Datenquellen

Bexio

  • Cloud Service, kein direktes Backup möglich

    • Die Daten werden täglich sichergestellt

    • Kopien davon sind auf mehreren Servern in der Schweiz (alle Backups in der Schweiz)

    • Ein Gesamtrestore ist jederzeit möglich

    • Bei Verlust einer Rechnung kann ein älterer Stand, dessen Zeitpunkt wir angeben, mitgeteilt werden (nicht auf DB restored, sondern Werte separat übermittelt)

    • Die Daten können jederzeit manuell in ein Excel exportiert werden

  • Kündigungsfall

    • Die Daten werden noch 40 Tage auf dem System belassen.

    • Danach wird ein Backup davon gezogen, der weitere 180 Tage behalten wird.

Google Drive

  • Cloud Service, kein direktes Backup möglich

    • Gelöschte Daten bleiben für 30 Tage im “Bin”

    • Files werden automatisch versioniert. Ältere Versionen können aber gelöscht werden, wenn sie nicht manuell auf “keep forever” gesetzt werden

    • Bei einem Ransomware-Angriff können Folder und Files um X-Tage zurückgesetzt werden, nicht aber ganze Drives

  • Automatische Snapshots durch Backup PC (stündlich), Google Docs werden nach Open Document Format konvertiert

  • TODO: Entfernen wenn Backup PC zuverlässig funktioniert: Monatlicher manueller Backup auf mfwinsrv01.

  • Zuständig: IT
