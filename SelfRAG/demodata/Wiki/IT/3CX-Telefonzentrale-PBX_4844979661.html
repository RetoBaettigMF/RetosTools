Importieren des Telefonbuches von vTiger

Der Import muss leider manuell gemacht werden. Eine CRM-Integration wäre möglich, würde aber Anpassungen im CRM bedeuten (z.B. keine Leerstellen in den Nummern). 

Vorgehen:

  • vTiger starten und Bericht "Telefonbuch Export für 3cx" auswählen: https://mf250.co.crm-now.de/index.php?module=Reports&view=Detail&record=230

  • CSV Export anklicken

  • Exportierte Datei mit Notepad++ öffnen und mit copy/paste in neue .csv Datei kopieren (Umgehungslösung, eigentliches Problem wurde nicht gefunden)

  • Bei der exportierten Datei muss die Kopfzeile geändert werden, z.B. mit Notepad++. Die Richtige Kopfzeile muss folgende sein:

    FirstName,LastName,Company,Mobile,Home,Business,Email

  • Folgende Texte ersetzen im ganzen File (mit Notepad++):

    • ",-" => ","

    • "'" => ""

    • " "" => """    (wiederholen, bis keine Vorkommen mehr gefunden wurden)

    • "" " => """    (wiederholen, bis keine Vorkommen mehr gefunden wurden)

Konfiguration der Extensions

Eventuell muss eine Extension zuerst aus dem Sipcall Trunk aktiviert werden (siehe Sipcall Provider)

Wichtig: Es können mehrere Extensions markiert und gemeinsam bearbeitet werden!

General

Mit dem QR-Code kann die Handy-App verbunden werden.

Mobile Number wird verwendet, wenn man parallel zum Klingeln des Telefons auch auf's Handy weiterleiten möchte und keine App hat.

Email ist wichtig und wird für die Zugangsdaten verwendet, um die App zu registrieren.

Web-Authentication: User hat auch über Webclient Zugriff aufs Telefon

  • Add a DID/DDI: Evtl. kann die DID hier direkt eingegeben werden, oder sie wird schon angezeigt, z.B. 41447474411 (411,411). Ansonsten siehe Direct Inbound Dialing (DID/DDI) weiter unten.

Voicemail

Standardmässig aktiviert. Nachricht wird als email versendet und gelöscht.

Ansage kann aufgenommen werden über User Interface:  Auf "Record" klicken, eindeutigen Namen vergeben, dann Extension wählen => Telefon klingelt. Abnehmen und Ansage aufnehmen.

Wichtig: Wav-File muss bestimmte Konventionen erfüllen: 1PCM, 8 kHz, 16 Bit, Mono, Max Size: 100MB, kann mit Audacity umgewandelt werden

Direct Inbound Dialing (DID/DDI)

  • Voice & Chat

  • Sipcall (Trunk)

  • DID Numbers

  • Zurück zu Users

  • User auswählen

  • Unter “Assigned DID number(s)” DID eingeben, z.B. 41447474411

Forwarding Rules

Es wird mit Stati gearbeitet. Es gibt 3 fix definierte Stati:

  • Verfügbar/Available

  • Abwesend/Away

  • Bitte nicht stören/Do not disturb

2 weitere können frei vergeben werden

Pro Status kann angegeben werden, was passieren soll. 

Die Stati können über das Web-Interface oder via App geändert werden. Der Status bezieht sich dann jeweils auf alle Telefone für diese Extension.

Wenn die Ansage leer ist, nimmt die Anlage jeweils die Standardansage der Extension (unter Voicemail eingestellt)

Phone Provisioning

Hier zuerst das Telefon auswählen (Snom oder 3cx App)

Einstellungen für Telefon können gemacht werden

Automatisch vergebene Passwörter nicht ändern

Rights

Jedem User können bestimmte Rechte gegeben werden, z.B. um Telefonbuch zu bearbeiten.

Admin-Account nicht rausgeben!

Sperrlisten (Blacklist)

Hat jemand 3x hintereinander das falsche PW eingegeben, wird seine IP-Adresse in die Sperrliste eingetragen. Sie muss dann vom Administrator manuell entfernt werden:

  • Login als admin

  • IP Sperrliste (links)

  • Adresse auswählen  und löschen

Digital Receptionist

Wird im Moment nicht verwendet, könnte für Feiertage etc. verwendet werden.

Es wird eine neue Extension erstellt, die dann bei den Inbound-Rules zugewiesen werden kann.

Könnte auch auf einen Status gelegt werden (z.B. benutzerdefinierter Status "Ansage" )

Ring Groups

Können direkt unter Inbound angesprochen werden oder von einer anderen Extension weitergeleitet werden.
