3. Präferenzen

Christian Ewald: Mercurial auf Bitbucket mit Jira und Confluence-Wiki, wichtig ist das Gesamtpacket von Versionskontrolle, Planungstools, Dokumentation (Wiki) und Continuous Integration

Markus Meier: Git auf Bitbucket mit Jira und Confluence-Wiki (TFS generell ok, aber CI könnte mit Webtechnologien problematisch sein)

Marcel Räber: Gesamtpacket muss stimmen, aber generell offen da keine Erfahrung mit Mercurial/Git vorhanden ist


4. Möglichkeiten

Durch Onlinerecherche und Erfahrung der Mitarbeiter wurden verschiedene Möglichkeiten zusammen getragen.

4.1 GitLab

Sehr Umfangreicher GitServer für verschiedene Linux-Plattformen. Bietet viele Zusatzfeatures wie Issuetracker, Milestones, Continuous Integration (mit Runner-Applikation für diverse Plattformen)

Kosten:  Gratis Version vorhanden, unterstützt nicht alle Features (z.B. keine Merge request approvals)

              Starter Version: 39$ pro Jahr und User

Reviews: 4.4/5 Sterne (112 Reviews) - Vor allem wird das Userinterface im Vergleich zu GitHub und Bitbucket bemängelt, aber das Produkt wird sehr schnell weiterentwickelt und sei sehr intuitiv und flexibel. Oftmals werden auch der günstige Preis und die gute Gratisversion gelobt.

4.2 TFS

TFS (Team Foundation Server) unterstützt seit Version 2013 Git-Repositories. Es sollten alle beliebigen Git-Clients zusammen mit TFS verwendet werden können. Ebenfalls bietet TFS Planungstools, Wiki und Continuous Integration Möglichkeiten.

Kosten:  TFS 2012 ist bereits vorhanden, für Git ist jedoch eine neuere Version nötig. Visual Studio-Abonnements umfassen eine Serverlizenz für TFS sowie eine TFS-CAL für den Abonnenten.

Upgrade: Upgrade to latest Version

Migration: Migration einer Collection auf einen anderen Server, TFCV zu Git

Reviews: 3.9/5 Sterne (69 Reviews) - Die Reviews sind meistens sehr positiv wenn TFS zusammen mit anderen Microsoft Diensten verwendet wird, negative Reviews finden sich überwiegend wenn TFVC und nicht Git als VCS eingesetzt wurde oder wenn TFS ausserhalb von Microsoft Diensten verwendet wurde (letzteres wurde auf einem lokal installierten TFS2018 ausprobiert: die Benützung der Git-Funktionen ist mit jedem beliebigen Client möglich (lokal getestet wurden: GitHub Desktop, GitKraken und TortoiseGit), die Planungs- und andere Tools sind im Webinterface (Webbrowser) vertreten und ausserdem wird seit 2017 "Team Explorer" als standalone Applikation angeboten.)

4.3 Bitbucket

Umfangreicher Server für diverse Plattformen. Unterstützt sowohl Mercurial als auch Git. Bietet viele Zusatzfeatures wie Issuetracking, Milestones, Burndown-Charts, Continuous Integration und Wiki wenn Bitbucket zusammen mit Jira, Confluence und ev. Jenkins verwendet wird.

Kosten:  BitBucketServer: 1800$ pro Jahr für 25 User; 3300$ pro Jahr für 50 User

              Confluence: 1500$ einmalig für 25 User; 2800$ einmalig für 50 User (bereits vorhanden -> cudoswiki)

              Jira: 2000$ einmalig für 25 User; 3600$ einmalig für 50 User (bereits als Cloudlösung vorhanden)

Reviews: 4.5/5 Sterne (102 Reviews) - Generell positiv, sei generell komplizierter als GitHub

4.4 GitHub Enterprice

Selbstgehosteter GitHub Server. Der Funktionsumfang ist praktisch identisch zu GitHub. Es besteht auch die Möglichkeit JIRA als Planungstool zu verwenden. GitHub Enterprice wird als virtual appliance ausgeliefert.

Kosten: 2500$ pro Jahr pro 10 User

Reviews: 4.7/5 Sterne (706 Reviews) - Sehr positiv

5 Reviews

Die Reviews stammen von G2Crowd. Folgende Abbildung stellt einen Vergleich der vier Möglichkeiten dar. Dabei ist anzumerken, dass für alle Lösungen ausser TFS ein gleichnamiger, gehosteter Service angeboten wird und die Reviews nicht spezifisch für die lokal gehostete Version gelten.


6 Trends

Mit Google Trends und Stackoverflow Trends wurden die zeitlichen Verläufe der Technologien verglichen.

Google Trends: Die Werte geben das Suchinteresse relativ zum höchsten Punkt im Diagramm im festgelegten Zeitraum an. Der Wert 100 steht für die höchste Beliebtheit dieses Suchbegriffs. Der Wert 50 bedeutet, dass der Begriff halb so beliebt war und der Wert 0 entspricht einer Beliebtheit von weniger als 1 % im Vergleich zum Höchstwert.

6.2 Bitbucket vs. GitLab vs. TFS vs. GitHub

Es wurden die Suchbegriffe "BitBucket", "GitLab", "Team Foundation Server" und "GitHub" verwendet. Die Darstellung von GitHub auf der Karte wurde ausgeblendet, da sie praktisch alles überdeckt. Anmerkung zu den Ergebnissen: Die grosse Differenz zwischen GitHub und den anderen Softwarelösungen entspricht den Erwartungen, da GitHub für sehr viele Community-Projekte (öffentlich zugänglich) eingesetzt wird und daher entsprechend oft in Suchanfragen erscheint.

(Karte ohne Github)

6.3 Stackoverflow Trends

Anmerkung: die Trends von Stackoverflow sind höchstwahrscheinlich von drei Faktoren anhängig: Grösse der Nutzercommunity, Anzahl auftretender Probleme (welche nicht "intuitiv" gelöst werden können) und Angebot von Support und Dokumentationen.

Git vs. Mercurial: Das Wachstum an Stackoverflow-Fragen zu Git korreliert mit dem Wachstum der Suchanfragen auf Google. Die Abflachung auf Stackoverflow könne drauf hindeuten, dass entweder die Benützung von Git einfacher geworden ist (z.B. durch Desktopapplikationen) oder gewisse Probleme gelöst wurden.

Softwarelösungen im Vergleich: Die Daten haben dieselben Tendenzen wie die Google-Anfragen, jedoch mit einem Ausreisser: TFS erfährt einen massiven Einbruch an Stackoverflow Fragen von 2009 bis 2012 und dies nicht, weil die Software stark an Nutzern verliert. Es könnte auf Probleme hindeuten, welche die damaligen TFS Versionen gehabt haben welche dann in folgenden Versionen behoben wurden.
