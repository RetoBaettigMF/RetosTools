IT : Umgang mit geheimen Daten

Created by Reto Bättig, last modified by nizami.sevindi on Aug. 26, 2024

1. Ziele

  • Sicherer Umgang mit geheimen Daten – Kundengeheimnisse und eigene vertrauliche Daten dürfen nicht  preisgegeben werden
  • Einhalten von Geheimhaltungsabkommen
  • Verschlüsselte Ablage von besonders geheimen Kunden- und Geschäftsdaten
  • Sicherstellung des Zugriffs auf sämtliche Daten und Systeme bei Ausfall von Mitarbeitern/GL-Mitgliedern

2. Grundsätze

  • Mitarbeiter sind prinzipiell (unabhängig von allfälligen NDAs) verpflichtet, mit vertraulichen Daten sorgfältig umzugehen und zu verhindern, dass Daten in falsche Hände geraten.
  • Falls vertrauliche Daten die Firma verlassen, sind diese zu verschlüsseln (Laptops, Smartphones, Emails, Memory-Sticks)
  • Vorsicht ist unterwegs geboten im Umgang mit vertraulichen Dokumenten und Telefongesprächen.
  • Geheimhaltungsvereinbarungen mit Kunden müssen den Projektmitarbeitern bekannt sein (Aufgabe des Projektleiters)
  • Für die militärische Geheimhaltung gelten spezielle Regeln, welche allen betroffenen Personen bekannt sein müssen (Aufgabe des Verantwortlichen für die militärische Geheimhaltung)

3. Methode zur Ablage von geheimen Daten

  • Zur Verschlüsselung von Daten wird firmenweit das Programm „Truecrypt“ bzw "Veracrypt" https://www.veracrypt.fr/en/Home.html verwendet. Die jeweils neueste Version erlaubt rückwärtskompatibel das entschlüsseln sämtlicher Container.
  • Vertrauliche Daten werden in einem verschlüsselten „File Container“ abgelegt.
  • Es sollen sichere Passwörter verwendet werden (gemäss aktuellem Stand der Technik)
  • Das Passwort für den Container muss einem GL-Mitglied mitgeteilt werden, welches das Passwort und den Speicherort des Containers sicher aufbewahrt.
  • Geheime Daten dürfen nicht unverschlüsselt auf mobilen Datenträgern (USB-Sticks, USB Harddisks etc.) gespeichert werden.

4. Methode zur Ablage von Login-Daten und anderen Geheimnissen, Tokens etc.

  • Falls irgendwo ein Konto eröffnet wird, welches für die Firma relevant sein kann, müssen Login-Name und Passwort so gespeichert werden, dass die berechtigten Mitarbeitenden und die GL Zugriff darauf haben und die Daten sicher aufbewahrt werden können.
  • Google Drive ist dank 2-Faktor Authentisierung sicher genug, um solche Daten zu speichern.

  • Für den Datenspeicher  existiert ein eigener Share auf Google Drive mit Namen "G:\Geteilte Ablagen\Secrets". Darauf Zugriff hat nur die Geschäftsleitung

  • Wenn man ein Passwort-File benötigt, wird auf diesem Share ein Word-File erzeugt (Mit Plain Text Files funktioniert das ganze nicht! Andere User können die Files nicht editieren). Diesem File werden dann gezielt die Benutzer und/oder zugewiesen, die Zugriff darauf haben müssen.  Beispiel: ""G:\Geteilte Ablagen\Secrets\#0060 Textrace Secrets.docx", Zugriff darauf hat nur die Gruppe textrace@cudos.ch

  • Ein Link zum Passwortfile wird abgelegt im Projektordner, z.B. unter "G:\Geteilte Ablagen\Projekte\TexTrace - Softwareentwicklung - #0060\002 Implementierung\5 Entwicklung\Software\#0060 Textrace Secrets.docx"

  • So findet man es einfach, es dürfen aber nur berechtigte Personen (mit 2-Faktor Authentisierung) darauf zugreifen

5. Sichere Ablage der Passwörter und Logins

  • Die GL führt eine sichere Ablage, in welcher die Logins und Passwörter gespeichert werden. Jedes GL-Mitglied kennt das aktuelle Passwort und die Speicherorte der Daten.
