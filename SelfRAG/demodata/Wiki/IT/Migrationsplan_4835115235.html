IT : Migrationsplan

Created by Reto Bättig, last modified by nizami.sevindi on Aug. 26, 2024
  • Sämtliche Daten mit DirSync auf Google Drive Kopieren, entsprechende DirSync Jobs erstellen.
    Die Jobs immer so definieren, dass auf GDrive eine exakte Kopie der lokalen Daten liegt (gelöschte lokale Daten auch auf GDrive löschen). => Master sind immer noch unsere lokalen Daten. Niemand darf auf den GDrive Daten arbeiten (Ausnahme: GL-Ordner)!
  • 2-Faktor Authentisierung durchsetzen
  • Schulung der Mitarbeitenden (an MA-Info März)
    • Installation von GDrive
    • Wie kann ich Daten offline bzw. schnell zur Verfügung haben?
    • Wie kann ich alte Versionen wieder holen?
    • Wie kann ich Versionen pinnen, auf die ich auch später noch zugreifen möchte?
  • Alle Mitarbeitenden installieren Google File Stream und schauen, ob sie die Verzeichnisse sehen => NOCH NICHT DARAUF ARBEITEN
  • Tag X: Umstellung machen (am Besten an einem Freitag Abend oder Samstag)
    • Fileshares der Verzeichnisse auf mfwinsrv01 entfernen => Verzeichnisse für MA nicht mehr sichtbar
    • Backup machen von allen Daten auf den Shares => auf separate Festplatte
    • Sämtliche Daten noch einmal synchronisieren auf GDrive (dauert hoffentlich dank der bereits vorsynchronisierten Daten nur noch 1-2 Stunden)
    • Verzeichnisse auf den Shares umbenennen in *_old
    • Dummy-Fileshares mit Symlinks zu neuen GDrive Verzeichnissen auf G:\ erstellen
    • Zugriffsrechte von alle@cudos.ch auf die Dateien von "Betrachter" auf "Content Manager" ändern (Ausser GL-Ordner)
    • Logon Skript für Domäne anpassen, damit Laufwerke nicht mehr gemounted werden: https://sid-500.com/2017/04/29/logon-skript-erstellen-active-directory/
    • Google Backup and Sync installieren auf mfwinsrv01
      • So konfigurieren, dass Daten wieder auf D:\data landen
      • Google lädt alle Daten runter von Cloud uns synchronisiert sie von nun an (dauert einige Stunden)
      • Backup von allen Google Drive Daten wird gleich wie früher erstellt ab mfwinsrv01
  • Info an alle Mitarbeitenden, dass auf sofort auf GDrive gearbeitet wird.
  • Migration der Daten von inStep nach G:\Geteilte Ablagen\Projekte
    • Ein Verzeichnis pro Projekt. Benennung:
      KUNDE - Projekt - Projektnummer
    • Migration der geschlossenen Projekte durch Hilfskraft => siehe Anleitung
    • Migration der aktiven Projekte durch die Projektleiter =>  siehe Anleitung
      • Der PL muss seinem Team sagen, wann er migriert und sicherstellen, dass das Team ab dann auf G: arbeitet und nicht mehr im inStep
