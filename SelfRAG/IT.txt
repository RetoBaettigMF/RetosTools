### Importieren des Telefonbuches von vTiger

Der Import muss leider manuell gemacht werden. Eine CRM-Integration wäre möglich, würde aber Anpassungen im CRM bedeuten (z.B. keine Leerstellen in den Nummern). 

Vorgehen:

1. vTiger starten und Bericht "Telefonbuch Export für 3cx" auswählen: <https://mf250.co.crm-now.de/index.php?module=Reports&view=Detail&record=230>
2. CSV Export anklicken
3. Exportierte Datei mit Notepad++ öffnen und mit Copy/Paste in eine neue .csv Datei kopieren. (Umgehungslösung, eigentliches Problem wurde nicht gefunden)
4. Bei der exportierten Datei muss die Kopfzeile geändert werden, z.B. mit Notepad++. Die richtige Kopfzeile muss folgende sein:
   ```
   FirstName,LastName,Company,Mobile,Home,Business,Email
   ```
5. Folgende Texte ersetzen im ganzen File (mit Notepad++):
   - `",-" => ","`
   - `' => ""`
   - ` " => """` (wiederholen, bis keine Vorkommen mehr gefunden wurden)
   - `"" " => """` (wiederholen, bis keine Vorkommen mehr gefunden wurden)
6. Im 3CX Webclient:
   - Admin
   - System
   - Phonebook
   - Bulk delete contacts
     - All contacts
   - Import
     - Import File auswählen

### Konfiguration der Extensions

Eventuell muss eine Extension zuerst aus dem Sipcall Trunk aktiviert werden (siehe [Sipcall Provider](Sipcall-Provider_8284733445.html))

Wichtig: Es können mehrere Extensions markiert und gemeinsam bearbeitet werden!

**General**

- Mit dem QR-Code kann die Handy-App verbunden werden.
- Mobile Number wird verwendet, wenn man parallel zum Klingeln des Telefons auch auf's Handy weiterleiten möchte und keine App hat.
- Email ist wichtig und wird für die Zugangsdaten verwendet, um die App zu registrieren.
- Web-Authentication: User hat auch über Webclient Zugriff aufs Telefon.

**Voicemail**

Standardmäßig aktiviert. Nachricht wird als Email versendet und gelöscht.

- Ansage kann aufgenommen werden über User Interface: Auf "Record" klicken, eindeutigen Namen vergeben, dann Extension wählen => Telefon klingelt. Abnehmen und Ansage aufnehmen.
- Wichtig: Wav-File muss bestimmte Konventionen erfüllen: 1PCM, 8 kHz, 16 Bit, Mono, Max Size: 100MB, kann mit Audacity umgewandelt werden.

**Direct Inbound Dialing (DID/DDI)**

- Voice & Chat
- Sipcall (Trunk)
- DID Numbers
- Add
- DID eingeben z.B. 41447474411
- Save
- Zurück zu Users
- User auswählen
- Unter “Assigned DID number(s)” DID eingeben, z.B. 41447474411
- Save

**Forwarding Rules**

Es wird mit Stati gearbeitet. Es gibt 3 fix definierte Stati:

- Verfügbar/Available
- Abwesend/Away
- Bitte nicht stören/Do not disturb

2 weitere können frei vergeben werden. Pro Status kann angegeben werden, was passieren soll. Die Stati können über das Web-Interface oder via App geändert werden. Der Status bezieht sich dann jeweils auf alle Telefone für diese Extension.

# Verhindern der Windows 10 Update Notification in Windows 7 & 8.1

Um zu verhindern, dass die Windows Update Notification wiederholt angezeigt wird, kann ein Windows Update deinstalliert werden. Es gibt mehrere Maßnahmen, um die Notification zu verhindern. Diese Methode ist am einfachsten (es müssen keine zusätzlichen Tools installiert werden) und hat für mich (BRS) funktioniert.

### Entfernen des Updates KB3035583

Um das Update zu entfernen, muss *Windows Update* in der Systemsteuerung geöffnet werden. In diesem Fenster muss unten links der Link *Installierte Updates* angeklickt werden.

Am einfachsten findet man das entsprechende Update, wenn man die Liste nach Namen sortiert und nach *Update for Microsoft Windows (KB3035585)* sucht.

Mit einem Rechtsklick auf das Update und der entsprechenden Bestätigung kann dieses deinstalliert werden. Danach muss ein Neustart durchgeführt werden.

### Update deaktivieren

Um zu verhindern, dass das Update automatisch wieder installiert wird, muss nach verfügbaren Updates gesucht werden. Wenn das Update KB3035583 erneut angezeigt wird, kann dieses mit einem Rechtsklick darauf und der Auswahl *Update ausblenden* inaktiviert werden.

### Automatisches Windows 10 Update

Vermehrt haben User davon berichtet, dass sich Windows 10 automatisch auf ihren Rechnern installiert hat. Laut Microsoft handelte es sich dabei um einen Fehler, bei dem das optionale Windows 10 Update per Default angekreuzt war. Dies wurde von Microsoft behoben. Es sollte deshalb nicht mehr vorkommen, dass sich das Windows 10 Update selbstständig installiert.

* [How to Geek](http://www.howtogeek.com/218856/how-do-you-disable-the-get-windows-10-icon-shown-in-the-notification-tray/)

# Anleitung: Sound Anlage

**Hinweis:** Wenn das Gerät **drahtlos mit dem Beamer verbunden** ist, funktioniert der **Sound automatisch** auch.

## Sound Anlage einschalten:

1. **Hauptschalter betätigen**  
2. **Lautstärke-Regeler einschalten mit der Fernbedienung**  
3. **Mixer einschalten (falls nicht automatisch an)**  
4. **Lautstärke einstellen und Audio abspielen**  
5. **Bei der drahtlosen Display Link Verbindung und beim Anschluss des Beamers via HDMI, wird der Ton jeweils automatisch über die Audio-Anlage abgespielt.**  
6. **Wenn die Verbindung zum Projektor via VGA erstellt wird, muss für das Abspielen von Audiodateien folgendes Kabel verwendet werden.**

## Ausführliche Anleitung zu den einzelnen Geräten

### 1. Setup

Die Anlage im Seminarraum SiZi_links setzt sich zusammen aus einem Funkmikrofon, einem Mixer und einer Lautsprecher Anlage. Für alle Komponenten sind Einstellungen vorgenommen worden, die grundsätzlich nicht verändert werden müssen. Betriebsanleitungen befinden sich im Regal im SiZi_links.

### 2. Empfänger

**Abbildung 1: Empfänger LD WSEC016 von LD Systems**

| 1 | Power Knopf | Mit einem Druck auf den Power Knopf wird der Empfänger eingeschaltet. |
|---|---|---|
| 2 | Kanal Auswahl | Mit den Kanal Auswahl Tasten kann ein Kanal zwischen 1 und 16 gewählt werden. Erst nach der Auswahl muss das Mikrofon neu gepaart werden. |
| 3 | Mikrofon Paarung | Nach der Auswahl eines Kanals wird die ASC-Taste gedrückt, um das Funkmikrofon an den Empfänger zu binden. |
| 4 | Volume | Mit dem Volume Drehregler kann die Ausgangslautstärke eingestellt werden. Übersteuerungen sollten vermieden werden. |

### 3. Mikrofon

**Abbildung 2: Belt Pack Sender mit Mikrofon**  
**Abbildung 3: Sender in der Draufsicht**

| 1 | Batterien | Der Sender enthält zwei Batterien des Typs „AA“. Nach dem Austausch müssen Sender und Empfänger neu verbunden werden. |
|---|---|---|
| 2 | IR Auge | Dieses Auge muss bei Kanalwechsel oder erneutem Pairing auf das Auge des Empfängers gerichtet sein. |
| 3 | Gain Regler | Der Gain Regler stellt das Gain-Signal ein. Es wird empfohlen, die Einstellung „Mic“ zu verwenden. |
| 4 | Ein/Aus Schalter | Die Mittelposition dient dem „Muten“ des Mikrofons. |

### 4. Mixer

**Abbildung 4: Mixer Alto Professional ZMX52**

| 1 | An/Aus Schalter | Hinten am Mixer befindet sich ein Schalter zum Ein- und Ausschalten. |
|---|---|---|
| 2 | Mikrofon Einstellungen | Anpassungen für die Mikrofon-Kanäle; Übersteuerungen werden mit einem roten LED „PEAK“ angezeigt. |
| 3 | PC Audio Einstellungen | Hier können Balance und Level für die Audioquellen eingestellt werden. |
| 4 | Main Lautstärke | Lautstärke für das gesamte System. Die „CLIP“ LEDs sollten nicht leuchten. |

### 5. Lautsprecher

**Abbildung 5: Edifier S730D Multimedia Speaker**

| 1 | Hauptschalter | Mit diesem Schalter kann die Anlage vom Strom genommen werden. |
|---|---|---|
| 2 | Lautstärkeregler | Hier kann die Lautstärke geregelt werden. Änderungen sollten idealerweise am Mixer vorgenommen werden. |
| 3 | Power/ Input Tasten | Mit der linken Taste kann die Anlage in Standby gesetzt werden. |
| 4 | Fernbedienung | Einstellungen an der Lautsprecheranlage können auch per Fernbedienung vorgenommen werden. |

### 6. Troubleshoot

| **Problem** | **Lösung** |
|---|---|
| Die Anlage ist stumm. | Überprüfen Sie, ob alle Geräte eingeschaltet sind und keine Mute-Funktion aktiviert ist. |
| Das Mikro funktioniert nicht. | Überprüfen Sie die Verbindung zwischen Mikrofon und Empfänger. |

# Einbindung externes VoIP-Telefon für ROU

## Zweck

Damit ROU von extern gegenüber Kunden mit einer Cudos Rufnummer (im Fall ROU: 044 747 44 68) kommunizieren kann, wurde ein separates VoIP-Telefon konfiguriert. Über diese Rufnummer kann ROU ein- und ausgehende Gespräche führen.

## Konfiguration

1. Beschaffung eines [Gigaset C530IP](https://www.digitec.ch/de/s1/product/gigaset-c530-ip-telefon-2738882) VoIP Telefons bei digitec.
2. In [sipcall business Konto](http://my.sipcall.ch/customer) folgendes sichergestellt:
   1. In der Rufnummerübersicht sicherstellen, dass das Verhalten bei eingehenden Anrufen bei der gewünschten Nummer auf "Klingeln, dann weiterleiten" gesetzt ist.
   2. In der Konfiguration ein neues VoIP-Passwort generieren (zwingend empfohlen, damit dieses vom Haupt-VoIP-Passwort abweicht!) und notieren.
   3. Allfällige Weiterleitung auf Alternativnummer hinterlegen (diese greift nach Ablauf der definierten Klingelzeit oder sofern das VoIP Telefon nicht aktiv ist).
3. Gigaset C530IP Funktelefon einrichten:
   1. sipcall als Provider aus Liste auswählen
   2. Anmeldename: 41447474468
   3. VoIP-Passwort: aus Schritt 2b bekannt.
4. Je nach Infrastruktur (Internetanbindung und Router mit NAT) ist noch eine Port Weiterleitung für die SIP-Signalisierung und RTP-Sprachpakete nötig. Für Gigaset Telefone sind diese typischerweise wie folgt:
   1. SIP: Port 5060/UDP auf Gigaset IP weiterleiten
   2. RTP Ports 5004-5020 auf Gigaset IP weiterleiten

Schritte 3 und 4 können einfacher als am Gerät selbst über das Webinterface des Gigaset C530ip gemacht werden. Die Bedienungsanleitung ist zu finden unter:  
G:\Geteilte Ablagen\Diverses\Administration\IT\Telefonanlage\Bedinungsanleitung Gigaset C530ip.pdf

**Hinweise:** Als guter Inbetriebnahme-Test eignet sich ein Anruf von einem Mobiltelefon auf die neu eingerichtete SIP-Nummer. So kann sichergestellt werden, ob die SIP-Signalisierung von außen durchkommt. Bei einem kurzen Gespräch lässt sich auch schnell feststellen, ob die RTP-Sprachdaten in beide Richtungen gehen (Stimmen auf beiden Endgeräten hörbar).

3. General
4. Boot Sequence: UEFI SanDisk Partition 1 ganz nach oben schieben

6. System Configuration
7. SATA Operation: AHCI
8. Apply
9. Secure Boot disablen

# Google Drive Anleitung für Mitarbeitende

## Installation und Verzeichnisstruktur

* [Die Installationsanleitung inkl. Verzeichnisstruktur ist hier zu finden](Google-Drive-Installation-und-Verzeichnissstruktur_4835115000.html)

## Arbeiten mit Google Drive

Google Drive hält normalerweise alle Daten in der Cloud und nicht auf der lokalen Festplatte. Dies ist nötig, weil wir insgesamt >1TB geteilte Daten haben und nicht alle diese Daten auf die lokale Festplatte synchronisieren wollen.

Solange man mit dem Internet verbunden ist und die Verbindungsgeschwindigkeit gut genug ist, lässt sich so gut arbeiten. Uploads werden zwischengespeichert und im Hintergrund auch über langsamere Leitungen vernünftig synchronisiert.

Daten, auf die man häufig zugreift, können jedoch bewusst auf der lokalen Festplatte gehalten werden, was auch den Zugriff darauf deutlich schneller macht.

### Daten offline verfügbar machen

Daten können nur online oder auch offline verfügbar sein. Welche Daten offline verfügbar sind, sieht man am Status-Icon:

* ![](attachments/4835115270/4835115287.png) Nur Online verfügbar
* ![](attachments/4835115270/4835115288.png?height=22) Im Moment am Synchronisieren
* ![](attachments/4835115270/4835115289.png?height=22) Offline verfügbar (und damit auch schnell)

Um Daten offline verfügbar zu machen, geht man wie folgt vor:

1. Rechtsklick auf den gewünschten Ordner (z. B. "Vorlagen").
2. Auf "Drive File Stream" => "Offline verfügbar" klicken.
3. Falls man später wieder lokal Speicherplatz freigeben möchte, kann man nicht mehr offline benötigte Ordner wieder auf "Nur online" setzen.

### Alte Versionen wiederherstellen

In Google Drive werden automatisch alle alten Versionen bis zu einem Alter von 30 Tagen gespeichert. Man kann jederzeit auf eine ältere Version zugreifen.

Der Zugriff erfolgt über das Web-Interface: [Google Drive](https://drive.google.com).

Hier navigiert man zum gewünschten File und macht einen Rechtsklick auf die Datei, um "Versionen verwalten" auszuwählen.

Im erscheinenden Fenster kann man alle verfügbaren älteren Versionen sehen, wer sie bearbeitet hat und wann. Über das Drei-Punkte-Menü kann man wählen, welche ältere Version man herunterladen möchte, um die aktuelle Version zu ersetzen.

### Wichtige Version für immer behalten

Ältere Versionen werden automatisch gelöscht, wenn sie älter als 30 Tage sind. Um eine bestimmte Version auch danach noch zu behalten, kann man die Version festhalten, indem man im Menü bei der entsprechenden Version auf "Niemals löschen" klickt.

### Datei finden

Die Online-Variante von Google Drive verfügt über eine sehr schnelle Volltextsuche. Dazu einfach im Suchfenster den gesuchten Begriff eingeben, um alle Dateien sofort zu sehen.

### Gelöschte Daten wiederherstellen

Gelöschte Daten werden für 30 Tage in den Papierkorb gelegt und können von dort wiederhergestellt werden. Dies funktioniert jedoch nur über das Web-Interface: [Google Drive Papierkorb](https://drive.google.com/drive/shared-drives).

### Große Datenmengen verschieben oder löschen

Da die Synchronisierung von Google File Stream nicht extrem schnell ist, funktionieren solche Aktionen sehr langsam, wenn man sie im Windows-Dateisystem macht. Viel schneller geht es, wenn man diese Funktionen über das Web-Interface durchführt: [Google Drive](https://drive.google.com/drive/shared-drives).

### Hinweis zu Spracheinstellungen

Unsere Anleitungen gehen alle davon aus, dass Deutsch als Systemsprache eingestellt ist. Wenn eine andere Sprache (z. B. Englisch) als Systemsprache eingestellt ist, funktionieren Links aus dem Wiki oder anderen Tools eventuell nicht richtig.

| Deutsch                      | Englisch                   |
|------------------------------|----------------------------|
| G:\Geteilte Ablagen\Admin     | G:\Shared drives\Admin      |
| G:\Meine Ablage                | G:\My Drive                |

## Telefon / Telco / Telefonkonferenz

Funktioniert immer, Audioqualität am besten, keine Video- und Bildschirmübertragung. Am besten geeignet für 1:1 Konferenzen. Oftmals die einfachste Methode, wenn man mit einem Tool anfängt und Audioprobleme hat: Tool nur für Video/Bildschirmübertragung verwenden und Audio über Telefon machen.

## Google Meet / Google Hangouts

Für alle Benutzer, die ein Google Konto haben, ist Meet/Hangouts sehr einfach zu verwenden. Hangouts ist das übergeordnete Messaging-Tool, welches auch Chats unterstützt. Meet ist das dazugehörige Videokonferenz-Tool.

Ein neuer Konferenzcall kann entweder direkt erstellt werden oder über Hangouts.google.com.

Teams ist bei Office365 dabei und benötigt eine separate Installation. In Teams kann ein Team und ein Kanal erstellt werden. Danach kann unten in der Mitte auf das Kamera-Icon geklickt werden, um eine neue Konferenz zu erstellen. 

### Troubleshooting

Wenn man in der Videokonferenz ist, kann über das Dreipunktemenü => Weitere Aktionen => Geräteeinstellungen Hilfestellung geleistet werden.

## Whereby

Zoom ist besonders gut geeignet für Online-Seminare. Man kann einfach bei Bedarf ein Abo lösen und es danach wieder kündigen. Zoom läuft am besten auf einem installierten Client unter Windows.

3. Die Datenbank `mf_website` in die Datenbank `testing_mf_website` kopieren

#### 2. Die aktuelle Webseite auf testing.m-f.ch kopieren

# 3CX Telefonzentrale Bedienung für Mitarbeitende

## Telefonieren vom PC aus

### Webclient

Die Benutzer erhalten von der IT eine E-Mail mit den Zugangsdaten (Extension Number, Pin, Passwort) sowie dem Link zum Webclient: [Webclient Link](https://mfengineering.3cx.ch:5001/webclient). Der Webclient funktioniert am besten mit Chrome.

### Einstellung der Audio-Geräte und Kamera

Lautsprecher, Mikrofon und Kamera können im Webclient unter **Einstellungen → Audio/Video** eingestellt werden.

### Paralleles Weiterleiten an Mobilgerät

Anrufe können an das Mobiltelefon weitergeleitet werden. Dafür:

1. Im Webclient unter **Einstellungen → Allgemein** die Mobilnummer eintragen (nur Zahlen, ohne Leerzeichen).
2. Unter **Einstellungen → Status** den Status anpassen.

### Spezialfunktionen

**Weiterleiten eines Anrufes**

Nach Annahme eines Anrufs kann dieser weitergeleitet werden:

- **Vermitteln (unattended):** Anruf wird direkt an eine Nummer weitergeleitet.
- **Makeln (attended):** Man ruft die andere Nummer an und verbindet den Anrufer beim Auflegen.
- **Konferenz:** Hinzunahme weiterer Anrufer.

**Annehmen eines Gespräches von einer fremden Nummer:**

Im Switchboard sieht man, wenn ein Telefon klingelt. Durch Klicken auf die Nummer kann man den Anruf annehmen.

### Chrome-Erweiterung

Für Chrome gibt es eine nützliche Erweiterung. Bei Installation kann man Telefonnummern im Browser direkt anklicken. Audio-Einstellungen sind unter dem "Hamburgermenu" → **Einstellungen → Personalisieren** zu finden.

### Telefonbuch

Das Telefonbuch wird regelmäßig von vTiger importiert. Es dürfen keine eigenen Adressen gespeichert werden; neue Nummern müssen im vTiger erfasst werden. Bei Bedarf kann die IT für eine Auffrischung des Telefonbuches kontaktiert werden.

### Fernsteuern von Geräten

Das Tischtelefon, Handy usw. kann „ferngesteuert“ werden, wenn das entsprechende Endgerät im Menü oben rechts ausgewählt wird:

- **Snom320:** Verbindung erfolgt über das Tischgerät.

**Atlassian Confluence CudosWiki Backup / Restore**

**Backup**

- Administration
- Cudoscloud / Benutzer verwalten
- Confluence
- Backup Manager
  - Backup für Server erstellen
    - Offline backup (um auf eigenem Server zurückzuladen)
  - Backup für Cloud erstellen

**Restore**

- Administration
- Cudoscloud / Benutzer verwalten
- Confluence
- Seite importieren (Site Import)

Der “Backup für Server” ist für den Restore auf einem lokalen Server gedacht.

Der “Backup für Cloud” ist für einen Restore in der Cloud gedacht. Dabei ist zu bedenken, dass in diesem Fall beim Restore die Gruppen (Rollen, Berechtigungen) NICHT restored werden (Stand 22.2.22). Der Grund ist Folgender: Wenn der Restore auf in einer anderen Destination (URL) gemacht wird, wäre das eine “Migration”, bei der die Berechtigungen eventuell anders sein müssten (Atlassian). Laut Atlassian soll es in zukünftigen Releasen eine Option “preserve group membership” geben, um beim Restore wieder die gleichen Berechtigungen zu haben wie vorher.

Um zur Zeit nach dem Restore die gleichen Berechtigungen zu haben gibt es zwei Möglichkeiten:

- Die Berechtigungen manuell wieder herstellen
- Den Umweg über einen lokalen Server machen:
  - Restore auf einen lokalen Server
  - Migration in die Cloud

### ACHTUNG:

- Da unsere Daten > 200MB sind, muss vor dem Restore zuerst bei Atlassian eine spezielle Bewilligung eingeholt werden, dass der Restore möglich ist.
- Mit einer free/trial ID kann zwar der Restore durchgeführt werden, die Daten (Bereiche / Spaces) werden aber nicht angezeigt. 
  - Umgehung: Trial upgrade production (14 Tage gültig)

## Konzept

Alle Daten des Unternehmens werden auf Google Drive gespeichert. Die Daten werden entweder von anderen Services gepusht oder der Backup Script Server lädt sie von den anderen Servern herunter. Das Kopieren kann auch manuell geschehen. Bei automatischen Lösungen muss ein Monitoring eingebaut werden (automatisch oder manuell).

Der Backup PC erzeugt periodisch (alle 1h, Minute '50) Snapshots auf Filesystem Ebene (btrfs). Diese sind jeweils mit einer Retention-Period versehen.

Der Backup PC muss möglichst autonom sein:
- Keine automatischen Updates
- Möglichst kein Zugriff von außen
- Firewall restriktiv einstellen

Falls Remote Administration verwendet wird:
- Zugriff nur von dedizierten Rechnern, welche sonst nicht verwendet werden. Z.Bsp: Altes Notebook.
- SSH mit Private Key. Key nur auf dediziertem Rechner und offline Speichermedium (z.Bsp. USB Stick)

**Dadurch soll verhindert werden, dass bei einer Kompromittierung der Haupt-IT die Backup-Server mit betroffen sind.**

Das Backup wird mit Hilfe von Influx/Grafana überwacht. Dies läuft auf einem Hetzner Server. Alle Vorgänge/Scripts senden an InfluxDB, wenn sie ausgeführt wurden, und Grafana Alerting-Rules überwachen, ob das auch stattgefunden hat.

**Source Code, inkl Dokumentation zum Aufsetzen:** [M_F Backup](https://dev.azure.com/MFEngineeringAG/_git/M_F%20Backup)

Zugriff:
- **Backup Monitoring und Scripts**
  - Influx: <https://backup.m-f.ch>
  - Grafana: [Grafana](https://backup.m-f.ch/grafana)
  - Unix user: root
  - Influx/Grafana: user: admin pwd: IT-Passwoerter
  - SSH-Key: it

- **Backup PC**
  - Steht beim Cudos Buero im Keller
  - user: user
  - pwd: user
  - Zugriff via lokalem Bildschirm/Tastatur

### Nickname ändern

In Discord kann man pro Server seinen Nicknamen ändern. Da wir unter uns sind und damit wir uns gegenseitig erkennen, ändere doch bitte deinen Nicknamen zu deinem Vornamen (pot. inkl. Nachnamen).

Dazu gehts du in Discord auf: Benutzereinstellungen > Profile > Serverprofile dort wählst du unter “Server Wählen“ den Server “Cudos Privat“, danach kannst du unter “Nicknamen“ deinen Vornamen eintragen.

![](attachments/8295153689/8295022656.png)### Rollen

In Discord gibt es Rollen, diese helfen den Server zu organisieren und die Rechte der Nutzer zu manangen.  
Im groben gibt es 3 Hauptrollen: Administrator, Mitarbeiter und Trainee.

Trainee wie auch Mitarbeiter haben die gleichen Berechtigungen.

Zusätzlich kann man sich noch selbst Rollen im Textchannel ”Subsricption” geben. Diese sind aber nur da um zu zeigen an was du Interesse hast. Momentan gibt es nur zwei dort, aber da kommen in Zukunft sicher noch mehr.

------------------------------------------

### Gästebuch

#### Rules-etc

#### Herzlich-Willkomme

Hier wird angezeigt wer gerade dem Server beigetretten ist. 

#### Subsricption

Hier kannst du dir selbst Rollen hinzufügen. Dafür musst du nur mit dem entsprechenden Emoji auf die Nachricht des Bots “Zira“ reagieren. Dadurch sehen die Anderen an was du interessiert bist und können alle mit dieser Rolle anpingen durch “@[Rollenname]“.

#### Inputs

Hier kannst du Anregungen zum Server reinschreiben. Wie der Wunsch nach neuen Text- wie Voicechannels, Bots etc. Bitte füge “@Administrator“ der Nachricht hinzu, damit die Admins die Nachricht auch sicher sehen.

### Pausenraum

Der Pausenraum beinhaltet Voicechannels welche frei genutzt werden dürfen und einen General Talk Textchannel um sich zu generellen Sachen auszutauschen.

### Freizeit-Aktivitäten

Hier gibt es Textchannels zu Freizeitaktivitäten, damit man sich z.B. zu den neusten Games austauschen kann oder zum Bouldern verabreden kann.

Hier kannst du dich mit den Anderen zu Themen Austauschen wie bspw. das neuste Dot NET 7 oder was ganz anderes. Auch hier bitte einfach neue Channels beantragen wie bei der Kategorie “Freizeit-Aktivitäten“.

Hier gibt es Textchannels zu diversen Themen wie IT-News etc. Zum Beispiel kannst du unter Food-Porn dein Cooles Weihnachtsessen mit den anderen teilen oder unter Haustiere ein herziges Foto deiner Katze oder deines Hundes.

### Zielsetzung

Das Ziel der Einführung einer HR-Software ist es, den Personalverwaltungsprozess zu optimieren und zu automatisieren. Durch den Einsatz einer HR-Software können Unternehmen ihre Personalabteilung effizienter gestalten, indem sie Aufgaben wie die Verwaltung von Mitarbeiterdaten, die Abwicklung von Lohnabrechnungen, die Verfolgung von Ferien- und Krankheitstagen sowie die Durchführung von Leistungsbeurteilungen vereinfachen. Eine HR-Software kann auch dazu beitragen, den Zugriff auf Informationen zu verbessern, die Kommunikation zwischen Mitarbeitern und der Personalabteilung zu erleichtern und die Einhaltung von Compliance-Vorschriften zu gewährleisten.

### .gitignore

Nicht immer sollen alle Dateien in einem Projektordner auch im git repository landen. Um Dateien aus der Versionsverwaltung auszuschließen, listet man sie in der `.gitignore` Datei.

Fragen, die sich stellen:

- **Dateiarten der jeweiligen Programmiersprache:**
  - Welche Dateien sind Quelldateien, welche Ergebnisdateien? z.B. `.py` vs. `.pyc`
  - Welche Ordner organisieren welchen Dateityp? Muss `/bin` mit verwaltet werden oder nicht? `/obj`?

- **Entwicklungsumgebung:**
  - Welche Dateien dienen der Projektorganisation und welche sind für persönliche Einstellungen bestimmt? z.B. `.suo`, `.sln` und `.user` Dateien von Visual Studio
  - Wie unterscheiden sich die Projektorganisationsstrukturen verschiedener Versionen einer Entwicklungsumgebung? z.B. `sqlite3\storage.ide` Datei in Visual Studio 2017

- **Eigenheiten eines Projektes:**
  - Sind bestimmte (Zwischen-)Ergebnisdateien nur sehr (zeit-)aufwendig zu generieren, können diese mit eingecheckt werden.
  - Sollen bestimmte persönliche Einstellungen im Projekt einheitlich sein, sind diese eventuell auch in die Versionsverwaltung mit aufzunehmen, z.B. einheitliche Codeformatierungsregeln, ReSharper templates, etc.

#### Nachträgliches Hinzufügen

Bei einem bestehenden Projekt ohne oder mit einer unpassenden `.gitignore` Datei kann man sich über die Seite [gitignore.io](https://gitignore.io/) eine passende Datei erstellen. Nach Eingabe des relevanten Kontexts (mit Autovervollständigung) erhält man die wichtigsten bzw. verbreiteten Regeln für die `.gitignore` Datei.

### Begriffe

**Absprungrate / Bounce Rate**

Die [Absprungrate](http://glossar.xeit.ch/bounce-rate) gibt den Prozentsatz der Besucher an, welche eine Landingpage nach dem Aufruf ohne Interaktion sofort wieder verlassen haben. Zur Absprungrate zählen also nur Sitzungen, wo eine *einzige* Seite aufgerufen worden ist.

![Absprungrate Grafik](http://blog.xeit.ch/wp-content/uploads/2016/04/Absprung_Website-520x178.png)

**Ausstiegsrate (% Ausstiege)**

Bei den Ausstiegen wird die *letzte Seite* einer Sitzung berücksichtigt, egal wie viele Seiten in der gesamten Sitzung zuvor aufgerufen worden sind. Die Ausstiegsrate ist also der Prozentsatz der Seitenaufrufe, welche die letzten einer Sitzung waren.

![Ausstiegsrate Grafik](http://blog.xeit.ch/wp-content/uploads/2016/04/Ausstieg_Website-520x83.png)

### Google Sheets Automatisierung mit AI und RolX

Für Google Sheets wurden Bibliotheken entwickelt, um Automatisierungen von Abfragen, Auswertungen etc. zu vereinfachen. Die Bibliotheken können im Apps Script eingebunden werden.

Entwickelt und gewartet werden die Bibliotheken momentan von [Reto Bättig](https://mfcloud.atlassian.net/wiki/people/557058:6697e3b1-b78b-46e9-8cec-69b2f592c15b?ref=confluence).

#### Cudos AI Library

- **Google Link:** <https://script.google.com/home/projects/11ctRQZSAFBVQpk9lynQOR9Ye1BjvtbvL3MfE_4LWE3y5sHVQCKI92Vih/edit> 
- **Script ID:** 11ctRQZSAFBVQpk9lynQOR9Ye1BjvtbvL3MfE_4LWE3y5sHVQCKI92Vih

**Funktionen:**

- **getCompletionAzure(args, deployment):**
  - Sendet eine Anfrage an die Azure OpenAI API, um eine Antwort zu erhalten.
  - Optional kann ein Deployment angegeben werden. Wenn nicht, wird ein Standard-Deployment verwendet.
  - Rückgabe: API-Antwort als JSON-Objekt oder Fehlermeldung.

- **getSingleCompletionAzure(prompt):**
  - Sendet einen einzelnen Prompt an die Azure OpenAI API und gibt die Antwort als Text zurück.
  - Protokolliert den Aufruf mit der `LogCall`-Funktion.

- **SheetGPTRange(prompt, range):**
  - Sendet eine Anfrage an die API und erwartet eine Antwort basierend auf einem Prompt und optionalen Daten in einem Bereich.
  - Versucht, die Antwort als JSON-Array zu parsen.
  - Rückgabe: Geparstes JSON-Array oder rohe Antwort.

- **SheetGPT(prompt, range):**
  - Sendet eine Anfrage an die API mit einem Prompt und optionalen Daten in einem Bereich.
  - Gibt die rohe API-Antwort zurück, ohne Parsing in ein JSON-Array.

#### Cudos Controlling Library

- **Google Link:** <https://script.google.com/home/projects/1zpZczeKyxaEmCRXqplbV0HTUj8rM8itfV0Jn9WXJr_4_5iUAcGQade9N/edit> 
- **Script ID:** 1zpZczeKyxaEmCRXqplbV0HTUj8rM8itfV0Jn9WXJr_4_5iUAcGQade9N

**Funktionsbeispiel:**

```javascript
function GetRolXData() {
  var daterange = CudosControllingLibrary.getDateRangeLastMonth();
  var data = CudosControllingLibrary.getRolXData(daterange);
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("Zeiterfassung");
  CudosControllingLibrary.clearSheetAndFillData(data, sheet);
}
```

#### Einbinden einer Library in Google Sheets

1. Im Google Sheets unter **Erweiterungen** > **Apps Script** die Script IDE starten.
2. Unter **Bibliotheken** auf das Plus-Symbol klicken.
3. Script-ID eingeben und auf **Suchen** klicken.
4. Auf **Hinzufügen** klicken.

### Migration einer Git-Repository

#### 1.2 - Erstellen eines Klones für die Migration

Der Klon für die Migration darf keine Arbeitskopie sein, sondern muss auch diejenigen Informationen enthalten, welche normalerweise nicht in einer lokalen Kopie enthalten sind. Dies kann mit dem Befehl

```
git clone --mirror
```

erreicht werden. Dabei wird die sogenannte „bare repository“ kopiert.

**Beispiel GitHub:**

```
$ git clone --mirror https://github.com/wannefab/Test.git
Cloning into bare repository 'Test.git'...
remote: Counting objects: 30, done.
remote: Compressing objects: 100% (16/16), done.
remote: Total 30 (delta 7), reused 30 (delta 7), pack-reused 0
Unpacking objects: 100% (30/30), done.
```

#### 1.2 - Erstellen einer leeren Repository auf dem Zielserver

Nun muss auf dem Server eine neue Repository angelegt werden. Diese soll komplett leer sein und z.B. nicht mit einem README initialisiert werden.

#### 1.3 - Klon auf den Zielserver laden

Für das Hochladen des Klones auf die Zielplattform muss wieder die „--mirror“ Option verwendet werden:

```
git push --mirror
```

**Beispiel mit GitHub:**

```
$ git push --mirror https://github.com/wannefab/Test.git
Counting objects: 30, done.
Delta compression using up to 4 threads.
Compressing objects: 100% (21/21), done.
Writing objects: 100% (30/30), 2.89 KiB | 118.00 KiB/s, done.
Total 30 (delta 7), reused 10 (delta 2)
remote: Resolving deltas: 100% (7/7), done.
To https://github.com/wannefab/Test.git
* [new branch]      master -> master
* [new branch]      secondBranch -> secondBranch
```

#### 2 mit TFS

Unter Code → 'RepositoryName' → Repository Importieren

## Probleme bei Google Drive

Bei GDrive kommt es manchmal zu Problemen, die folgendermassen behoben werden können:

* [Langsames Öffnen von Dateien in Office](#ProblemebeiGoogleDrive-LangsamesÖffnenvonDateieninOffice)
* [Komplette Rechnerabstürze beim Arbeiten mit Office](#ProblemebeiGoogleDrive-KompletteRechnerabstürzebeimArbeitenmitOffice)
* [Files sind auf G: nicht sichtbar oder andere Probleme](#ProblemebeiGoogleDrive-FilessindaufG:nichtsichtbaroderandereProbleme)

### Langsames Öffnen von Dateien in Office

Manchmal gibt es Probleme, mehrere Dokumente in Microsoft Office zu öffnen, welche auf GDrive liegen. Grund scheint das Office-Plugin für die Zusammenarbeit zu sein.  
Falls ihr Probleme habt, deaktiviert bitte die Echtzeitzusammenarbeit:

1. Bei Taskliste auf Gdrive Icon rechtsklicken => Gdrive Fenster kommt
2. Im GDrive Fenster auf Zahnrad klicken => Einstellungen
3. Echtzeit-Anwesenheit in Microsoft Office disablen
4. OK klicken => GDrive wird neu gestartet und Probleme sollten behoben sein.

### Komplette Rechnerabstürze beim Arbeiten mit Office

Es ist schon mehrfach vorgekommen, dass beim Arbeiten mit Word (insbesondere beim Drucken) der Rechner komplett eingefroren ist. Wenn dieses Problem auftritt, ist es meistens auch reproduzierbar, d.h. es kann gar nicht mehr vernünftig mit dem Rechner gearbeitet werden.

In diesem Fall hilft es, wenn man den Link auf GDrive aus den Vorlagenverzeichnissen im Word entfernt. Dies geht oft nur noch über die Registry, weil Word schon beim Einstellen der Vorlagenordner abstürzt.

Folgende Registry-Keys manuell entfernen:  
`[HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Common\General]` => "SharedTemplates" den String mit GDrive löschen.

### Files sind auf G: nicht sichtbar oder andere Probleme

Zuerst überprüfen, ob sie im Browser sichtbar sind (unter <https://drive.google.com/>).

Falls ja, Cache bei GDrive folgendermassen löschen:

1. Drive FS stoppen:
   - Bei der Taskleiste auf die Programm-Icons gehen und Gdrive rechtsklicken.
   - **Wichtig: Zuerst warten, bis alle Files synchronisiert sind, sonst können Daten verloren gehen (unten muss "Alle Dateien sind auf dem aktuellen Stand" stehen).**
   - Unter Einstellungen auf "Beenden" klicken.
2. Cache löschen: `%USERPROFILE%\AppData\Local\Google\DriveFS`
3. Gdrive neu starten (Windows-Taste => Google Drive App).
4. Neu einloggen in Google Drive.
5. => Jetzt synchronisiert Gdrive neu und alles sollte wieder funktionieren.

SAP JCo ist eine Schnittstelle, die Java mit einem SAP System verbindet. Es erlaubt Aufrufe in beide Richtungen (Java → SAP und SAP → Java).

Um JCo nutzen zu können, braucht man die Dateien `sapjco3.jar` und `sapjco3.dll`. Diese müssen in den Projektordner verschoben werden.

Um eine Verbindung zu SAP aufzubauen, braucht man eine Implementation des Interface `com.sap.conn.jco.ext.DestinationDataProvider`. Dieses hat die Methoden `getDestinationProperties`, `setDestinationDataEventListener` und `supportEvents`.

Die letzten zwei Methoden sind für einfache Anwendungen unwichtig, man kann sie leer lassen bzw. immer `false` zurückgeben lassen.

`getDestinationProperties(String destName)` hat als Rückgabewert ein `java.util.Properties` Objekt, das die Eigenschaften der Verbindung mit Namen `destName` beschreibt. Dabei sind die Eigenschaften `client`, `user`, `ashost`, `passwd`, `sysnr` notwendig für den Aufbau einer Verbindung.

Einen `DestinationDataProvider` kann man nun registrieren mit der Zeile

```
Environment.registerDestinationDataProvider(destinationDataProvider);
```

und mit 

```
JCoDestination dest = JCoDestinationManager.getDestination(destName);
```

eine Verbindung instanzieren. WICHTIG: diese Verbindung wird noch nicht bei der Instanzierung geöffnet, sondern erst, sobald man einen Aufruf macht.

Zum Abrufen von Informationen aus dem SAP braucht es sogenannte "Function Modules". Ein Function Module kann man mit SAP JCo erzeugen mit dem Befehl

```
JCoFunction function = dest.getRepository().getFunction(moduleName);
```

Dabei ist der String `moduleName` der Name des Function Modules (z.B. `"GETCUSTOMERDATA"`).

Parameter für diese Funktion (hier z.B. die ID oder Namen des Kunden) kann man über `function.getImportParameterList().setValue(String paramName, String value)` setzen.

Eine Funktion ausführen kann man mit `function.execute(dest)`.

Falls die Funktion Rückgabewerte hat (hier die Kundendaten), erhält man diese mit `function.getExportParameterList().getXX(String paramName)`. Dabei ist `XX` der DatenTyp, den man für die Rückgabewerte haben will (`int`, `String`, `double` etc.).

**Links:**

- [JCo API 3.0](http://www.finereporthelp.com/download/SAP/sapjco3_linux_32bit/javadoc/overview-summary.html)
- [Anleitung für eine JCo Verbindung mit Erklärung (sehr hilfreich)](http://www.softserv.at/sap-java-connector-verbindung-herstellen/)
- [Offizielle SAP Seite zu JCo](https://help.sap.com/saphelp_nwpi711/helpdata/de/46/fb807cc7b46c30e10000000a1553f7/content.htm)

### SSL Zertifikate

Für die bei METANET gehosteten Webseiten können über Plesk einfach kostenlose Let's Encrypt SSL Zertifikate erstellt werden. Damit ist der Datentransfer zwischen Server und Client über HTTPS auch verschlüsselt.

#### Wissenswertes zu Let's Encrypt Zertifikaten

- Let's Encrypt Zertifikate können jeweils nur für 1 Domain respektive Subdomain erstellt werden. Es handelt sich also nicht um Wildcard-Zertifikate, und daher sind unter Umständen mehrere Zertifikate zu erstellen.
- Die Zertifikate sind 3 Monate gültig, werden jedoch durch Plesk automatisch erneuert.
- Bevor das Let's Encrypt Zertifikat aktiviert werden kann, muss die DNS-Verwaltung bei der neuen Domain deaktiviert sein.

#### Vorgehen zur Aktivierung eines Zertifikates

Die Aktivierung von Let's Encrypt Zertifikaten muss zwangsweise mit dem Hauptbenutzer **mfchbae** erfolgen. Ansonsten kommt eine Fehlermeldung.

1. Bei der gewünschten Domain/Subdomain auf den Let's Encrypt Link klicken.
2. Im Folgescreen die Benachrichtigungs-Emailadresse auf [admin@cudos.ch](mailto:admin@cudos.ch) setzen. Sofern es sich um eine Hauptdomain (z.B. cudos.ch) handelt, macht es Sinn, das Zertifikat auch mit dem Alternativnamen inkl. "www." aufzunehmen (über Checkbox).
3. Nach einem Klick auf "Installieren" erscheint eine Bestätigungsmeldung. Der Webserver wird danach automatisch neu gestartet. Nach ca. 10 Minuten ist die Website über HTTPS erreichbar.

#### Automatische Umleitung von HTTP auf HTTPS

Um sicherzustellen, dass eine Website beim Aufruf über verschlüsselte Kommunikation geladen wird, ist eine automatische Umleitung von HTTP auf HTTPS nötig. Dies kann durch Erstellen einer `.htaccess`-Datei im Root-Verzeichnis der jeweiligen Domain erfolgen mit folgendem Inhalt:

```plaintext
RewriteEngine On
RewriteCond %{SERVER_PORT} !^443$
RewriteRule (.*) https://%{HTTP_HOST}/$1 [L]
```

## Ziele

* Sicherer Umgang mit geheimen Daten – Kundengeheimnisse und eigene vertrauliche Daten dürfen nicht preisgegeben werden.

## Grundsätze

* Mitarbeiter sind prinzipiell (unabhängig von allfälligen NDAs) verpflichtet, mit vertraulichen Daten sorgfältig umzugehen und zu verhindern, dass Daten in falsche Hände geraten.
* Falls vertrauliche Daten die Firma verlassen, sind diese zu verschlüsseln (Laptops, Smartphones, Emails, Memory-Sticks).
* Vorsicht ist unterwegs geboten im Umgang mit vertraulichen Dokumenten und Telefongesprächen.
* Geheimhaltungsvereinbarungen mit Kunden müssen den Projektmitarbeitern bekannt sein (Aufgabe des Projektleiters).
* Für die militärische Geheimhaltung gelten spezielle Regeln, welche allen betroffenen Personen bekannt sein müssen (Aufgabe des Verantwortlichen für die militärische Geheimhaltung).

## Methode zur Ablage von geheimen Daten

* Zur Verschlüsselung von Daten wird firmenweit das Programm „Truecrypt“ bzw. "Veracrypt" verwendet. Die jeweils neueste Version erlaubt rückwärtskompatibel das Entschlüsseln sämtlicher Container.
* Vertrauliche Daten werden in einem verschlüsselten „File Container“ abgelegt.
* Es sollen sichere Passwörter verwendet werden (gemäss aktuellem Stand der Technik).
* Das Passwort für den Container muss einem GL-Mitglied mitgeteilt werden, welches das Passwort und den Speicherort des Containers sicher aufbewahrt.
* Geheime Daten dürfen nicht unverschlüsselt auf mobilen Datenträgern (USB-Sticks, USB Harddisks etc.) gespeichert werden.

## Methode zur Ablage von Login-Daten und anderen Geheimnissen, Tokens etc.

* Falls irgendwo ein Konto eröffnet wird, welches für die Firma relevant sein kann, müssen Login-Name und Passwort so gespeichert werden, dass die berechtigten Mitarbeitenden und die GL Zugriff darauf haben und die Daten sicher aufbewahrt werden können.

* Für den Datenspeicher existiert ein eigener Share auf Google Drive mit Namen "G:\Geteilte Ablagen\Secrets". Darauf Zugriff hat nur die Geschäftsleitung.

Diesem File werden dann gezielt die Benutzer und/oder zugewiesen, die Zugriff darauf haben müssen. Beispiel: "G:\Geteilte Ablagen\Secrets\#0060 Textrace Secrets.docx", Zugriff darauf hat nur die Gruppe [textrace@cudos.ch](mailto:textrace@m-f.ch).

* So findet man es einfach, es dürfen aber nur berechtigte Personen (mit 2-Faktor Authentisierung) darauf zugreifen.

### IT: Unscharfe Schriften an Laptops mit Full-HD Bildschirmen

**Ursache**

Die hohe Auflösung resp. die hohe DPI (Dots per Inch) Rate an den Full HD (1920x1080) Laptop Bildschirmen wird von Windows automatisch erkannt. Zur "Verbesserung" der Benutzerfreundlichkeit wird dazu eine Vergrösserung eingeschaltet. Da die meisten Applikationen diese Vergrösserungsfunktion aber nicht unterstützen, resultiert dies in unscharfen Schriften und verwaschenen Grafiken. Ebenfalls existieren Applikationen, welche diese Funktionalität nur teilweise oder gar fehlerhaft unterstützen. Dies kann zu verzerrten und abgeschnittenen GUI-Elementen führen.

**Lösung**

Die automatische Vergrösserung aller GUI Elemente kann ausgeschaltet werden. Dazu muss die Systemeinstellung *Display* (Deutsch: *Anzeige*) geöffnet werden.

In dem geöffneten Menü kann mit einem Klick in das rot markierte Kontrollkästchen (Deutsch: *Manuell eine Skalierungsstufe für alle Anzeigegeräte auswählen*) die Anzeige umgeschaltet werden.

Nun kann die Vergrösserung mit einem Klick auf den obersten Radiobutton (Deutsch: Kleiner - 100%) ausgeschaltet werden und mit "Apply" (Deutsch: Übernehmen) bestätigt werden.

Dass die Änderungen übernommen werden, ist ein erneuter Login erforderlich.

### IT: Upgrade auf Windows 10

**Möglichkeiten**

- Unsere Windows 7 oder Windows 8/8.1 Lizenzen können gratis auf Windows 10 updated werden. (Hinweis: ab 29. Juli 2015 bietet Microsoft das Windows 10 Upgrade für 1 Jahr gratis an!)
- Ein Upgrade geschieht über eine laufende Windows Installation und funktioniert in der Regel problemlos - d.h. nach dem Upgrade kann direkt weitergearbeitet werden.
- Wer von Windows 7 auf Windows 10 umsteigen will und ein Clean Install (d.h. Windows komplett frisch aufsetzen) plant, der muss trotzdem zuerst ein Upgrade des laufenden Windows 7 machen. Dabei wird die Windows 7 Seriennummer auf dem Aktivierungsserver aktiviert und mit der Hardware verknüpft. Erst danach ist ein Clean Install möglich.

**Vorgehen**

**Variante A (für Windows 8/8.1 Benutzer)**

- Windows 10 über Windows Taskleisten-Symbol reservieren -> sobald es verfügbar wird (kann Stunden/Tage dauern, da Microsoft stufenweise Downloads freischaltet), kann das Upgrade eingespielt werden.

**Variante B (für Windows 7/8/8.1 Benutzer, schneller)**

- ISO-Image auf Netzlaufwerk nehmen und mit z.B. [Rufus](http://rufus.akeo.ie/) einen bootable USB-Stick (mind. 8GB) erzeugen.
- Alternativ: ISO mit ISO-Tool (z.B. [PowerISO](http://poweriso.com/)) mounten und Upgrade installieren.

**Wichtige Hinweise**

- Wer virtuelle Maschinen basierend auf Oracle **VirtualBox** benutzt, soll mit dem Upgrade noch zuwarten -> VMs können mit der aktuell verfügbaren Version (5.0.1) unter Windows 10 nicht gestartet werden!

IT : Zweite MS Teams Instanz

Man kann mittels einem Script eine zweite Microsoft Teams Instanz öffnen. Somit kann man beispielsweise eine Instanz mit dem Account der Partnerfirma offen haben und eine Instanz mit dem Account der Cudos. Somit bleibt einem erspart, Teams im Browser auszuführen.

### Anleitung

1. Versichere dich, dass Teams offen ist.
2. Kopiere folgenden Code ins Notepad.
3. Speichere das Text File z.B. als 'M_F_Teams.bat'.
4. Starte das Script (per Doppelklick auf das .bat File).
5. Logge dich im Teams mit dem gewünschten Account ein.

Zum Öffnen der zweiten Instanz jeweils das Script starten, nachdem die erste Instanz offen ist.

Anmerkung: Du musst dich dann nicht jedes Mal einloggen.

```batch
@ECHO OFF

REM Uses the file name as the profile name
SET MSTEAMS_PROFILE=%~n0
ECHO - Using profile "%MSTEAMS_PROFILE%"

SET "OLD_USERPROFILE=%USERPROFILE%"
SET "USERPROFILE=%LOCALAPPDATA%\Microsoft\Teams\CustomProfiles\%MSTEAMS_PROFILE%"

REM Ensure there is a downloads folder to avoid error described at
REM https://gist.github.com/DanielSmon/cc3fa072857f0272257a5fd451768c3a
mkdir "%LOCALAPPDATA%\Microsoft\Teams\CustomProfiles\%MSTEAMS_PROFILE%\Downloads"

REM Fixes missing "Desktop" folder
mkdir "%LOCALAPPDATA%\Microsoft\Teams\CustomProfiles\%MSTEAMS_PROFILE%\Desktop"

ECHO - Launching MS Teams with profile %MSTEAMS_PROFILE%
cd "%OLD_USERPROFILE%\AppData\Local\Microsoft\Teams"
"%OLD_USERPROFILE%\AppData\Local\Microsoft\Teams\Update.exe" --processStart "Teams.exe"
```

Hier der Link zum GitHub Gist: 

<https://gist.github.com/DanielSmon/cc3fa072857f0272257a5fd451768c3a>
